    Using ORNL Supercomputers

    • Summit will be replaced by Frontier based on AMD CPU and GPU
    • Need to switch from CUDA to AMD HIP Introduction to HIP Programming Guide
    • This could happen in next 6-8 months
    • We discussed ways to get accounts which are suitable for different size and different length projects
    • Collaboration with ORNL staff is the way we operate now
    • Porting /exploring as we do can be supported by a Director’s Discretionary for a standalone MLCommons project
    • The description and link for the Director's Discretionary (DD) allocation is at the link https://docs.olcf.ornl.gov/accounts/accounts_and_projects.html
    • Large projects (not MLCommons) get ALCC (Leadership Computing Challenge) or Insight awards
    • Ask Feiyi Wang

    Papers

    • Benchmark Carpentry paper could be combined with another paper Gregor is writing for Frontiers journal on HPC; this discusses power use
    • Christine discussed power paper and asked what type of network is Candle UNO. Geoffrey thought it was a database like lookup mappring drug specifications to biological responseJuri is porting a PNNL single GPU code to many GPUs.
    • We hoped to have a draft of this paper by the next meeting

    New Benchmarks

    • We discussed removing data engineering components of benchmarks such as earthquake one.
    • We discussed OSMIBench with its new open dataset based on OpenFOAM and TensorFlowFoam (Argonne GitHub - argonne-lcf/TensorFlowFoam: Integrating the TensorFlow 1.15 C-API into OpenFOAM 5.0 for data-driven CFD algorithm development)
    • we will discuss this UVA ORNL May 12
    • Wes Brewer mentioned intereste in Digital Twins of Supercomputers – in particular Frontier to study Power. He will talk in detail next time. There will be an SC23 BOF
    • Christian mentioned that FastML was collecting datasets for their Physics datasets. 3 have been fully formed. A student of Vijay Janapa Reddi and Bonn University are involved.
